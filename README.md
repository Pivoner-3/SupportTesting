# SupportTesting

Техническое задание на создание консольного приложения 
для вычисления простейших арифметических задач.

Цель технического задания:
Цель разработки – создание функционального и надежного инструмента для быстрого выполнения математических расчетов. Реализация данного ТЗ позволит автоматизировать рутинные вычисления, повысить эффективность работы пользователей, нуждающихся в простых математических операциях и предоставить основу для дальнейшего расширения функциональности калькулятора. Важно обеспечить корректную обработку ошибок ввода и предотвратить возникновение исключительных ситуаций, которые могут привести к некорректной работе программы. Разработка должна соответствовать современным стандартам кодирования на C# и обеспечивать простоту поддержки и модификации в будущем. В рамках ТЗ не предусматривает разработка графического интерфейса или интеграция с внешними сервисами. 

Функциональные требования:
1.	Ввод чисел. Программа должна запрашивать у пользователя ввод двух чисел.
2.	Сложение. Должна быть реализована операция сложения введенных чисел.
3.	Вычитание. Должна быть реализована операция вычитания введенных чисел.
4.	Умножение. Должна быть реализована операция умножения введенных чисел.
5.	Деление. Должна быть реализована операция деления введенных чисел.
6.	Обработка ошибок. Должна быть реализована базовая обработка ошибок. Например: деление на ноль, некорректный ввод типов данных и т. д.
 Дополнительные требования, не обязательные к реализации:
1.	Возведение в степень. Должна быть реализована операция возведения в степень введенного числа.
2.	Извлечение корня. Должна быть реализована операция извлечения квадратного корня введенного числа.

Нефункциональные требования:
1.	Программа должна иметь минимальное время отклика на запрос.
2.	Программа должна быть написана с использованием принципов чистого кода и поддерживать стандарты кодирования C#.
3.	Приложение должно корректно работать в среде .NET 6 или выше.
4.	Код должен быть хорошо документирован (комментарии, описание функций).
5.	Программа должна быть устойчива к случайным ошибкам и сбоям при некорректном вводе.
6.	Минимальные требования к ресурсам системы (память, процессорное время).
7.	Реализовать логирование ошибок для упрощения отладки.
